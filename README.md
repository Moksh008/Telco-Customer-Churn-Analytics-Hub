#  Telco Customer Churn Analytics Hub ⚡

Customer Churn Prediction System is a premium, industry-grade customer retention management dashboard designed to predict subscriber churn, analyze risk factors, and empower support representatives with automated, data-backed retention toolkits.

The system is powered by a high-performance **Random Forest Classifier** trained on the standard Telco Customer Churn dataset, utilizing a serialized preprocessing **StandardScaler** to map and scale 19 distinct customer demographic, service, and contract features.

---

## 🔗 Notebook Reference
The machine learning modeling, exploratory data analysis (EDA), feature engineering, and model validation were conducted in the following Kaggle notebook:
👉 **[Moksh Kaggle Notebook - UL Project](https://www.kaggle.com/code/moksh72/moksh-2410998600-s-ul-project)**

---

## 🌟 Dashboard Modules & Navigation Guide

The platform is structured into four distinct high-performance modules, accessible via the left-side navigation panel:

### 1. 🏠 Welcome & Project Overview (Landing Page)
- **Purpose**: Serves as the executive gateway to the Churn Hub platform.
- **Use Case**: Provides context on the underlying business problem, explains the machine learning pipeline, and details how to utilize each predictive dashboard tab to maximize subscriber lifetime value.

### 2. 👤 Single Customer Predictor & "What-If" Simulator
- **Purpose**: Real-time customer assessment for customer support and sales representatives.
- **Use Case**: Input an individual subscriber's characteristics (contract type, payment method, tenure, monthly charges, and digital bundles). The model immediately calculates their churn risk and displays a visual color-coded risk gauge.
- **Explainable AI Features**: Automatically pinpoints the customer's **Key Risk Drivers** (e.g. month-to-month contracts, lack of digital bundles).
- **Targeted Retention Toolkit**: Dynamically suggests custom retention strategies based on their specific vulnerabilities.
- **"What-If" Simulator**: Allows agents to mock contract extensions, autopay registration, or monthly discounts to see *exactly* how the churn risk probability decreases *before* pitching the solution to the client.

### 3. 📁 Enterprise Batch CSV Predictor
- **Purpose**: Automated, high-throughput bulk customer scoring for executive campaigns.
- **Use Case**: Download our standardized CSV input template, upload a customer list (hundreds of rows), and compile prediction results instantly.
- **Analytics Display**: Features global metrics (total scored, predicted churn rate, average risk) and beautiful, interactive Plotly charts showing risk distribution histograms and scatter analyses (Tenure vs. Monthly Charges by Churn Risk).
- **Exporting**: Enables downloading the complete scored dataset, containing raw features, exact churn probabilities, binary predictions, and risk categories.

### 4. 📊 Model Global Insights & Explainability
- **Purpose**: Strategic, high-level analysis for business managers and marketing directors.
- **Use Case**: Renders the Random Forest's mathematical **Feature Importances** in an interactive horizontal bar chart, showing which parameters carry the most weight globally. Summarizes concrete executive actions to structurally prevent churn.

## 🛠️ Data Pipeline & Feature Mappings

Through rigorous reverse-engineering of `scaler.pkl` and `churn_model.pkl`, the dashboard perfectly aligns all raw user selections with the model's expected 19-feature input vector:

### Encoded Categories (Alphabetical Ordinal Encoding)
1. **gender**: `Female` (0), `Male` (1)
2. **SeniorCitizen**: `No` (0), `Yes` (1)
3. **Partner**: `No` (0), `Yes` (1)
4. **Dependents**: `No` (0), `Yes` (1)
5. **PhoneService**: `No` (0), `Yes` (1)
6. **MultipleLines**: `No` (0), `No phone service` (1), `Yes` (2)
7. **InternetService**: `DSL` (0), `Fiber optic` (1), `No` (2)
8. **OnlineSecurity**: `No` (0), `No internet service` (1), `Yes` (2)
9. **OnlineBackup**: `No` (0), `No internet service` (1), `Yes` (2)
10. **DeviceProtection**: `No` (0), `No internet service` (1), `Yes` (2)
11. **TechSupport**: `No` (0), `No internet service` (1), `Yes` (2)
12. **StreamingTV**: `No` (0), `No internet service` (1), `Yes` (2)
13. **StreamingMovies**: `No` (0), `No internet service` (1), `Yes` (2)
14. **Contract**: `Month-to-month` (0), `One year` (1), `Two year` (2)
15. **PaperlessBilling**: `No` (0), `Yes` (1)
16. **PaymentMethod**: `Bank transfer (automatic)` (0), `Credit card (automatic)` (1), `Electronic check` (2), `Mailed check` (3)

### Continuous Variables (Standardized via StandardScaler)
17. **tenure**: Float, customer tenure in months (0–72).
18. **MonthlyCharges**: Float, monthly subscription rate ($18.25–$118.75).
19. **TotalCharges**: Float, cumulative subscription charges ($18.80–$8684.80).

## 🚀 Installation & Local Execution Guide

To set up the project and run the dashboard locally on a Windows machine:

### 1. Prerequisites
Ensure you have Python 3.10 to 3.14 installed on your machine.

### 2. Setup Virtual Environment
Open PowerShell or Command Prompt in the project folder and run:
```powershell
# Create a virtual environment
python -m venv venv

# Activate virtual environment
.\venv\Scripts\activate
```

### 3. Install Dependencies
```powershell
pip install -r requirements.txt
```

### 4. Run the Streamlit Dashboard
Launch the web server by executing:
```powershell
streamlit run app.py
```
This will automatically compile the application and launch it at: **`http://localhost:8501`**
